CSS Locators

This is just an additional set of CSS selector challenges, for you to strengthen even more your skills. There is no need to resolve these from a testing framework.

You don't know how to test your selectors within your browser? try out:

  • Finding Xpath and CSS in Chrome Developer Tools
  • Chrome Plugin: CSS and XPath checker

Recommended training materials:
Web Element Locator Strategies. You can also search on google for css selectors reference and you will find plenty learning materials.

Exercises

Given the HTML in the right, resolve the following challenges

  • Part 1: Basic selectors
    1. Select all elements with class="intro"
    2. Select the element with id="Lastname"
    3. Select all elements with class="intro", and the element with id="Lastname" (with one single selector).
    4. Select all <h1> elements.
    5. Select all <h1> elements and all <p> elements (with one single selector).
    6. Select all <p> elements that are inside a <div> element.
    7. Select all <p> elements where the parent is a <div> element.
    8. Select the <p> element that is next to a <ul> element.
    9. Select all <table> elements that are siblings of a <ul> element.
    10. Select all elements.
    11. Select all <p> elements with class="myquote".
  • Part 2: Attribute selectors
    1. Select all elements with an id attribute.
    2. Select all elements with an id attribute value equal to "my-Address"
    3. Select all elements with an id attribute value ending with "ess"
    4. Select all elements with an id attribute value equal to "my" or starting with "my" followed by a hyphen (-)
    5. Select all elements with an id attribute value starting with the letter "L"
    6. Select all elements with a title attribute value containing the word "beautiful"
    7. Select all elements with an id attribute value containing the string "s"
    8. Select all <p> elements with a lang attribute value starting with "it"
  • Part 3: Pseudo classes for inputs
    1. Select all checked form elements.
    2. Select all disabled form elements.
    3. Select all enabled form elements.
    4. Select the input element that currently has focus (reload you browser in case that autofocus is not currently active).
    5. Select all <input> elements where the value is invalid according to their limitations.

    6. Select all <input> elements where the value is valid according to their limitations.

  • Part 4: Children selectors
    1. Select all <p> elements that are the first child of their parent.
    2. Select all <p> elements that are the first <p> element of their parent.
    3. Select all <p> elements that are the last child of their parent.
    4. Select all <p> elements that are the last <p> element of their parent.
    5. Select all even <tr> elements.
    6. Select all odd <tr> elements.
    7. Select all <li> elements that are the first child of their parent, counting from the <li> element.
    8. Select all <li> elements that are the second <li> element of their parent.
    9. Select all <b> elements that are the only child of their parent.
    10. Select all <h3> elements that are the only child of its type, of their parent.
